# AstrBot 微博监控插件 (Weibo Monitor)

定时监控微博用户动态并推送到指定会话。  
由 Trae 配合 Gemini-3-Flash-Preview 模型开发。  
可以 Star 一下吗，求求你了。🥺🥺🥺

## 功能特性

- **定时监控**：自定义检查频率，支持间隔随机化避免被反爬。
- **多用户支持**：可同时监控多个微博账号，支持URL、UID、用户名三种输入格式。
- **定向推送**：支持为每个微博账号单独配置允许接收推送的群聊/私聊会话 ID。
- **精准推送**：仅推送最新更新，自动过滤置顶微博。
- **媒体开关**：可分别控制是否推送微博图片、微博视频，以及是否额外附加网页截图。
- **转发媒体识别**：转发微博中的原微博图片、视频也会被识别并按规则推送。
- **分步发送**：先发送正文消息，再按配置将图片或视频以组合转发方式单独发送。
- **自动清理缓存**：下载的图片与截图会写入插件缓存目录，并自动清理过期缓存文件。
- **Cookie 必填**：必须配置微博 Cookie 才能正常抓取数据，确保稳定性。
- **灵活过滤**：支持屏蔽词过滤，可选择是否推送原创/转发微博。
- **消息自定义**：支持自定义推送消息格式，满足不同需求。
- **配置导入导出**：支持配置的导入导出，方便迁移和备份。
- **数据持久化**：自动持久化监控数据，重启后不会重复推送历史微博。
- **错误恢复**：内置请求重试和异常处理机制，保证监控稳定性。

## 部署步骤

1. **安装**：
   在 AstrBot 插件市场安装本插件。
   插件会自动安装 `requirements.txt` 中的依赖。如果手动安装，请在服务器环境下运行：
   ```bash
   pip install httpx beautifulsoup4
   ```

2. **获取会话 ID (unified_msg_origin)**：
   在你想接收推送的会话（群聊或私聊）中，向机器人发送指令：
   ```
   /get_umo
   ```
   机器人会返回当前会话的 ID，请记录下来。

3. **配置插件**：
   在 AstrBot 管理面板 -> 插件设置 -> `weibo_monitor` 中进行配置：
   - `认证信息 -> 微博 Cookie`: **必填项**，必须填写才能正常使用。
   - `微博账号与推送范围 -> 订阅规则`: 每新增一条规则，就填写：
     - `微博主页 / UID / 用户名链接`: 微博用户主页链接、UID 或用户名链接
     - `允许接收推送的会话 ID`: 允许接收这条微博推送的会话 ID，支持逗号或换行分隔
     - 如果 `允许接收推送的会话 ID` 留空，则该微博账号默认不推送给任何会话
   - `推送内容 -> 消息模板`: 自定义推送消息格式，支持 `{name}`、`{weibo}`、`{link}` 变量。
   - `推送内容 -> 推送微博图片`: 控制是否下载并推送微博原图。
   - `推送内容 -> 推送微博视频`: 控制是否推送微博视频。
   - `推送内容 -> 推送原创微博 / 推送转发微博`: 控制推送微博类型。
   - `推送内容 -> 屏蔽关键词 / 必含关键词`: 控制正文过滤规则。
   - `截图设置`: 控制是否额外附加网页截图以及截图参数。
   - `运行设置`: 控制检查频率与请求间隔。

## 自定义消息格式

你可以在插件设置的 `message_format` 中自定义推送消息的样式。支持以下变量：
- `{name}`: 微博用户的昵称。
- `{weibo}`: 微博正文内容（已自动处理换行）。
- `{link}`: 微博原文链接。

**默认格式示例：**
```
🔔 {name} 发微博啦！

{weibo}

链接: {link}
```

## 常用指令
- `/get_umo`: 获取当前会话 ID（用于配置推送目标）。
- `/weibo_verify`: 验证当前设置的 Cookie 是否有效。
- `/weibo_check`: 立即抓取列表里第一个账号并推送最新一条微博。
- `/weibo_check_all`: 立即抓取列表里所有账号并推送最新一条微博。
- `/weibo_export`: 将当前插件的所有配置导出为 Base64 字符串。
- `/weibo_import [配置字符串]`: 使用导出的配置字符串快速恢复或迁移配置。
## 重要说明

### 推荐配置方式
推荐按后台分区依次配置：

1. 在 `认证信息` 中填写微博 Cookie。
2. 在 `微博账号与推送范围 -> 订阅规则` 中新增规则。
3. 在 `微博主页 / UID / 用户名链接` 填入一个微博账号。
4. 在 `允许接收推送的会话 ID` 填入允许接收该账号更新的群聊或私聊 ID。

这样每个微博账号都能单独控制推送到哪些会话，逻辑最直观；如果规则里的会话 ID 留空，则该账号不会推送。

### 发送顺序说明
推送顺序已调整为：

1. 先发送一条普通消息，内容为 `消息模板 + 微博正文 + 链接`。
2. 如果开启了 `推送微博图片` 或 `推送微博视频`，再将媒体内容以组合转发形式单独发送。
3. 如果开启了 `附加微博网页截图`，截图会附加在第一条正文消息中。

### 配置结构变更说明
从当前版本开始，插件**不再兼容旧版平铺配置**，例如旧的 `weibo_urls`、`target_conversation_id` 等字段已移除。

升级后请在后台按新的分区配置重新填写：
- `认证信息`
- `微博账号与推送范围`
- `推送内容`
- `截图设置`
- `运行设置`

### Cookie 必填提醒
⚠️ **微博Cookie为必填项**，插件虽然允许在未配置 Cookie 时启动，但**必须配置 Cookie 后才能正常抓取数据**。请务必按照下方步骤获取并配置Cookie。

## 如何获取微博 Cookie

1. 在电脑浏览器打开 [微博移动端官网](https://m.weibo.cn/) 并登录。
2. 按 `F12` 打开开发者工具，切换到 `网络 (Network)` 选项卡。
3. 刷新页面，在左侧列表中找到第一个 `m.weibo.cn` 的请求（或者任何一个 `getIndex` 请求）。
4. 在右侧的 `请求标头 (Request Headers)` 中找到 `Cookie` 字段。
5. 复制该字段的完整值，粘贴到插件设置的 `认证信息 -> 微博 Cookie` 中。

### Cookie 注意事项
- Cookie具有有效期，失效后需要重新获取
- 请勿在多设备同时登录同一账号，可能导致Cookie失效
- 获取Cookie时请确保使用微博移动端官网 (m.weibo.cn)，而非PC端官网
- 配置完成后可使用 `/weibo_verify` 指令验证Cookie是否有效

## 常见问题

**Q: 为什么配置了Cookie还是抓不到数据？**

A: 请先使用 `/weibo_verify` 指令验证Cookie是否有效。如果Cookie有效但仍无法抓取，请检查：
1. 微博URL格式是否正确
2. 检查间隔是否设置过短导致被反爬
3. 查看日志是否有其他错误信息

**Q: 为什么会重复推送相同的微博？**

A: 正常情况下插件会自动记录最后推送的微博ID，不会重复推送。如果出现重复推送，可能是：
1. 插件数据文件损坏或丢失
2. 微博ID发生变化（极少情况）
3. 可以使用 `/weibo_check` 指令手动检查是否重复

**Q: 如何提高监控稳定性？**

A: 建议：
1. 检查间隔设置为10分钟以上
2. 启用间隔随机化（在 `运行设置` 中调整检查间隔随机浮动和请求间隔随机浮动）
3. 定期更新Cookie，避免失效
4. 监控账号数量不宜过多
